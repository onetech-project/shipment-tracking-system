# Quickstart: Modern Dashboard UI Revamp

**Feature**: `003-modern-dashboard-ui`  
**Audience**: Developers implementing or reviewing this feature  
**Branch**: `003-modern-dashboard-ui`

---

## Overview

This feature revamps the `apps/frontend` rendering layer only. The backend
(`apps/backend/`) and shared packages (`packages/shared/`) are **not modified**.

The revamp:
1. Adds **Tailwind CSS** and **shadcn/ui** to `apps/frontend`
2. Extracts shared UI primitives into `src/components/shared/` and `src/components/layout/`
3. Refactors all pages and feature components to use Tailwind utility classes (removing all
   inline `style` props)
4. Makes the dashboard layout fully responsive (mobile hamburger + Sheet drawer)
5. Adds 5 new Playwright E2E spec files covering responsive behaviour

---

## Prerequisites

- Node.js ≥ 18 (existing requirement)
- The backend must be running (or mocked) for E2E tests
- Existing project dependencies already installed (`packages/shared`, `apps/backend`)

## Step 1: Install Tailwind CSS and shadcn/ui

```bash
cd apps/frontend

# Install Tailwind v4 + PostCSS + animation plugin
npm install -D tailwindcss@^4.2 postcss autoprefixer tailwindcss-animate

# Initialise shadcn/ui (interactive — accept all defaults; choose "slate" base colour)
npx shadcn@latest init

# Add the required shadcn/ui components
npx shadcn@latest add button input label card badge table dialog sheet \
    dropdown-menu form separator avatar
```

After running these commands, the following files are created/modified:
- `tailwind.config.ts` — generated by shadcn init, then updated per contracts/design-tokens.md
- `postcss.config.mjs` — generated by shadcn init
- `components.json` — shadcn project config (see contracts/design-tokens.md §5)
- `src/app/globals.css` — replace content with the token definitions from contracts/design-tokens.md §3
- `src/components/ui/` — shadcn component files (auto-generated, do not hand-edit)
- `src/lib/utils.ts` — `cn()` helper function (generated by shadcn)

## Step 2: Create Layout Components

Create the following files in `src/components/layout/`:

```
src/components/layout/
├── sidebar.tsx           # Desktop persistent nav
├── mobile-drawer.tsx     # Sheet-based mobile nav
├── top-bar.tsx           # Mobile top bar with hamburger
└── dashboard-shell.tsx   # Composes the above + main content area
```

See `contracts/shared-components.md §1` for prop interfaces.

The `DashboardShell` component manages sidebar open/close state via a local `useState`.
It renders:
- `<Sidebar />` inside a `hidden lg:flex` wrapper (desktop only)
- `<TopBar onMenuToggle={toggle} />` inside a `flex lg:hidden` wrapper (mobile only)
- `<MobileDrawer open={isOpen} onClose={close} />` — Sheet rendered but hidden when closed
- `<main>` content area with `flex-1 overflow-auto p-4 lg:p-8`

## Step 3: Update the Dashboard Layout

Replace the inline-style layout in `src/app/(dashboard)/layout.tsx`:

```tsx
// BEFORE
<div style={{ display: 'flex', minHeight: '100vh' }}>
  <Navigation />
  <main style={{ flex: 1, padding: '2rem' }}>{children}</main>
</div>

// AFTER
<DashboardShell>{children}</DashboardShell>
```

The auth guard logic (`useAuth` redirect) remains unchanged inside `layout.tsx`.

## Step 5: Revamp Feature Components

## Step 6: Add Responsive Playwright Tests

Create 5 new spec files in `apps/frontend/e2e/ui-responsive/`:

```
e2e/ui-responsive/
├── navigation.spec.ts      # User Story 1: hamburger, sidebar, drawer
├── design-system.spec.ts   # User Story 2: hover states, focus rings, modal animation
├── dashboard-home.spec.ts  # User Story 3: welcome card, ActionCard grid
├── upload-mobile.spec.ts   # User Story 4: upload flow at 375px viewport
└── settings-mobile.spec.ts # User Story 5: users table scrollable at 768px
```

**Viewport configuration** (per test file):
```typescript
test.use({ viewport: { width: 375, height: 812 } });  // mobile
test.use({ viewport: { width: 768, height: 1024 } }); // tablet
test.use({ viewport: { width: 1280, height: 800 } });  // desktop
```

**Key test patterns**:
```typescript
// Navigation - hamburger visible on mobile
await expect(page.locator('[aria-label="Open menu"]')).toBeVisible();

// Sidebar permanent on desktop
test.use({ viewport: { width: 1280, height: 800 } });
await expect(page.locator('nav[data-sidebar]')).toBeVisible();

// Modal animation — check visible after trigger
await page.click('button:has-text("Inactivate")');
await expect(page.locator('[role="dialog"]')).toBeVisible();

// Table scroll on mobile — no horizontal overflow
const table = page.locator('.overflow-x-auto');
await expect(table).toBeVisible();
```

## Common Pitfalls

| Pitfall | Solution |
|---------|---------|
| Tailwind classes not applied | Ensure `src/app/globals.css` has `@import "tailwindcss"` at the top |
| shadcn components not found | Run `npx shadcn@latest add <component>` — components must be explicitly added |
| `cn()` not found | Ensure `src/lib/utils.ts` is generated by `shadcn init` |
| Animations not suppressed in CI | Use `motion-safe:` prefix on all transition classes; CI environments don't set `prefers-reduced-motion` so tests should pass |
| data-testid missing after revamp | Cross-check against the testid inventory in `contracts/shared-components.md §3` |
| Sheet flicker on desktop | Ensure `lg:hidden` wrapper prevents Sheet from rendering on desktop |
| Inline style prop ESLint warning | Remove all `style={{…}}` — replace with Tailwind utility classes |

## Architecture Decision Record

| Decision | Rationale |
|----------|-----------|
| Tailwind CSS v4 | Current release; native CSS variable integration; no `tailwind.config.js` for tokens |
| shadcn/ui (CLI copy model) | Components owned by the project; no runtime CDN dependency; fully customisable |
| Sheet for mobile drawer | Built-in a11y (focus trap, Escape close); Radix-based; consistent with design system |
| No Framer Motion | YAGNI; Tailwind `transition-*` utilities suffice for all specified animations |
| No TanStack Table | Tables are simple static grids; YAGNI; DataTable wrapper provides all required features |
| `motion-safe:` prefix | Zero-JS reduced-motion support; Tailwind built-in; covers FR-007 / SC-007 |
